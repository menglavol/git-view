## 1. 安装与首次启动

安装步骤见 [README](../README.md#安装)。首次启动时：

- **Git 环境检测**：GitView 依赖系统 Git CLI。若未检测到可用 Git，应用会进入**引导页**，按你的操作系统给出安装命令（macOS 用 Homebrew/Xcode、Windows 官网下载、Linux 包管理器）。装好后点「已安装，重新检测」即进入主界面；若 Git 装在非标准位置，点「去设置手动指定路径」在设置里指定可执行文件路径。
- **凭据存储检测**：账号 Token 存放于操作系统原生密钥库。Linux 桌面若提示「凭据存储不可用」，请安装 `gnome-keyring` 或 `libsecret` 后重启。

主界面为三段式布局：左侧导航(首页 / 远程仓库 / 本地仓库 / Clone 中心 / 账号管理 / 操作日志 / 设置)、顶栏(当前账号、同步、设置入口)、主内容区。**首页仪表盘**聚合展示账号数、远程/本地仓库数、运行中克隆数、有未提交变更的仓库数，以及最近打开仓库、最近任务和快捷入口。

## 2. 账号管理

进入「账号管理」页，点「添加账号」。

### 2.1 添加账号

1. **选择平台**：GitHub / GitLab.com / 私有 GitLab / Gitee。
2. **填写 Token**：在对应平台生成 Personal Access Token（需含读取仓库、按需写入的权限），粘贴到 Token 输入框（输入框为密码态，不缓存）。
3. **私有 GitLab 额外配置**(选择「私有 GitLab」时展开)：
   - **Web 地址**：如 `https://git.company.com:8443`，GitView 会自动推导 API 地址 `https://git.company.com:8443/api/v4`（可手动修改）。
   - 是否允许 **HTTP 明文**(内网场景，勾选时会弹安全提示)、是否允许**自签名证书**(仅对该实例生效)、使用系统代理 / 自定义代理、请求超时、默认 Clone 协议、SSH 主机别名、API 路径前缀。
4. **测试连接**：点「测试连接」，成功后显示用户名与头像；**只有测试通过后「保存」才可用**。

### 2.2 管理账号

账号列表每行可见头像、平台、用户名、服务地址、是否默认、是否启用、最近同步时间，以及操作按钮：

- **设为默认**：默认账号用于各页面的默认筛选；只能把**已启用**的账号设为默认。
- **启用 / 禁用**：禁用账号后其远程仓库缓存保留但不再参与同步与默认筛选；若禁用的是当前默认账号，系统会自动把默认转移到第一个已启用账号并提示你。
- **测试连接 / 同步 / 编辑**。
- **删除**：触发危险操作确认对话框，需输入用户名关键词二次确认。删除会**清除系统密钥库中的凭据**，但**保留已克隆的本地仓库记录**。

## 3. 远程仓库浏览与搜索

进入「远程仓库」页。

- **同步仓库**：点顶部「同步」，可选「同步当前账号」或「同步全部账号」。同步分页拉取并显示进度(如「已同步 30/50」)。**同步失败不会清空旧缓存**，会提示「已保留上次缓存」。
- **筛选**：按平台、账号、Owner、可见性(public/private)、仅收藏、仅已克隆组合筛选。
- **搜索**：搜索框输入关键词，对仓库名与描述实时过滤(已缓存数据 500ms 内)。
- **收藏**：点收藏图标，收藏状态跨重启持久化。
- **详情**：点仓库名或「详情」打开右侧抽屉，分「基本信息」「提交历史」两个标签。
  - **基本信息**：展示仓库名、描述、平台、所属账号、Owner、默认分支、可见性、Web/HTTPS/SSH 地址、语言、最近更新时间；若已克隆还显示本地路径。可一键复制 HTTPS / SSH 地址、打开网页。
  - **提交历史**：分页加载该远程仓库的提交(GitHub / GitLab / Gitee 通用)，滚动到底自动加载下一页；点击某条提交展开**提交详情**——完整提交信息、作者 / 时间 / 短 SHA、增删行数汇总，以及改动文件列表(新增 / 修改 / 删除 / 重命名标签 + 每文件增删数)，每个文件可展开查看着色 diff，并可「打开网页」跳到平台对应提交页。

列表数据量大时自动启用虚拟滚动，5000+ 条不卡顿。

## 4. 批量克隆

这是 GitView 的核心能力。

1. 在「远程仓库」列表中**多选**若干仓库，点「批量 Clone」。
2. 在配置对话框中设置：
   - **目标根目录**：点选择按钮挑选目录。
   - **目录组织方式**：
     - 扁平：`<根>/<仓库名>`
     - 按 Owner：`<根>/<owner>/<仓库名>`
     - 按平台和账号(默认)：`<根>/<平台>/<owner>/<仓库名>`，如 `~/Projects/github/torvalds/linux/`
   - **Clone 协议**：HTTPS / SSH(默认取自设置)。
   - **并发数**：1–8，默认 3。
   - **克隆完成后自动加入本地仓库**(默认开)。
   - **克隆分支**：每个仓库预览行有分支下拉，默认选中该仓库的默认分支；展开下拉时按需从平台拉取完整分支列表(可搜索)，选择后克隆时用 `git clone --branch <分支>` 只检出该分支。分支列表拉取失败时回退为仅默认分支，不影响克隆。
   - 对话框底部预览每个仓库的最终目标路径。
3. 确认后跳转「Clone 中心」。

在「Clone 中心」：

- 顶部统计：总数 / 运行中 / 成功 / 失败 / 跳过 / 已取消。
- 每个任务显示仓库名、目标路径、状态、进度百分比与当前阶段(如 `Receiving objects: 65%`)、错误信息。
- 操作：**取消**(终止子进程并清理半成品目录)、**重试**(失败任务重新入队，不影响其他)、打开目录、移除。
- 顶部可「全部启动」，以及分状态「清空已完成 / 已失败 / 已取消」（三个按钮各自清理，删除前二次确认，仅删任务记录、不影响磁盘文件）。
- 若目标目录已存在同名仓库，该任务标记为 **skipped**。

> 安全：HTTPS 克隆私有仓库时，Token 通过临时凭据机制注入，**不会写入 `remote.origin.url` 或日志**。克隆完成后 `git remote get-url origin` 是干净的 URL。

## 5. 本地仓库管理

进入「本地仓库」页。列表按各仓库本地路径的**目录结构以树形组织**：目录为可展开 / 折叠的父节点（**默认折叠**），仓库为叶子；「子仓库」列显示该目录下的仓库数，目录行另以小标签汇总其下各状态计数。若某目录本身是仓库、其下又有子仓库，会**合并为一个可展开的仓库节点**（名称右侧标注子仓库数）。仓库名 / 本地路径过长会省略，鼠标悬浮显示完整内容。

- **添加仓库**：选择一个已存在的 Git 仓库目录纳入管理。
- **扫描目录**：选择父目录(如 `~/Projects`)批量识别其中所有 `.git` 仓库(默认深度限制，遇到 `.git` 即识别且不深入，避免扫到子模块)，跳过已纳入的。
- **状态总览**：每行显示当前分支、远程地址、状态标签、未提交文件数、ahead/behind 计数。状态包括 clean / 已修改 / 需推送 / 需拉取 / 无远程 / 分离头指针 / 冲突 / 路径丢失。
- **刷新所有状态** / **批量 Fetch**：并行处理多个仓库，单个失败不阻塞其余，结果汇总展示。
- **打开仓库**(进入工作区)、**打开目录**、**打开终端**。
- **从列表移除**：触发危险操作确认，强调**仅从 GitView 列表移除、不删除磁盘文件**。

500+ 仓库虚拟滚动，状态刷新不阻塞界面。

## 6. 单仓库 Git 工作流

在「本地仓库」点「打开仓库」进入工作区(三栏布局)。

- **顶栏**：**返回（回本地仓库列表）**、仓库名、当前分支(分支选择器)、ahead/behind、远程地址，以及 Fetch / Pull / Push、打开目录 / 终端按钮。
- **左侧 · 文件变更**：分「已暂存」「未暂存」两段，每行带状态字母(M/A/D/R/U/C)与路径。
  - 单击切换 stage / unstage；支持「全部暂存」「全部取消暂存」与状态过滤。
  - **丢弃变更(discard)**：右键 → 危险操作确认对话框 → 列出将丢弃的文件与「不可恢复」警告 → 输入仓库名二次确认后才执行。
- **中间 · Diff 查看器**：新增行绿底、删除行红底。大文件(>1MB)与二进制 / 图片显示占位提示，建议用外部工具查看。
- **右侧 · 提交面板**：填写 message(必填)与可选 description。提交按钮在「无暂存文件 / message 为空 / 未配置 user.name/email / 处于冲突或分离头指针」时禁用并提示原因。
- **分支选择器**：列出本地分支与远程分支(`origin/` 前缀)。
  - **脏工作区保护**：存在未提交变更时，切换分支按钮禁用，悬浮提示「请先提交或暂存后再切换分支」；新建分支不受此限制。
  - 可「从远程分支创建本地分支」(自动设置 upstream)、「新建分支」(可选是否切换过去)。
- **提交历史**：分页显示 hash 短码、message、作者、日期，滚动到底自动加载下一页；点击某条提交展开**提交详情**——完整提交信息、作者 / 时间 / 短 SHA、增删汇总，以及改动文件列表与每个文件可展开的着色 diff(与远程仓库「提交历史」共用同一详情视图)。

常见网络操作提示：

- **Pull 冲突**：提示用外部工具(VS Code / `git mergetool`)解决后再提交。
- **Push 被拒(非快进)**：提示「远程有新提交，请先 Pull」。
- **Push 无 upstream**：提供「创建 upstream 分支」按钮(`git push -u origin <branch>`)。
- **Push 权限不足**：提示确认 PAT 是否含写权限。

> **策略与身份即时生效**：在「设置 → Git」配置的默认 **pull 策略**(fast-forward / rebase / merge)、默认 **push 策略**(simple / current / upstream)以及 **user.name / user.email**，保存后立即作用于本页的 Pull / Push / Commit —— pull 按所选策略执行(如 rebase 时用 `git pull --rebase`)、push 按所选 `push.default` 执行、提交时若仓库无本地身份则用设置中的 user.name/email 自动补写(仓库已配置则不覆盖)。同理，「设置 → Git」指定的 **Git 可执行文件路径**对所有 Git 操作(fetch/pull/push/commit/checkout 等)生效。

## 7. 操作日志与诊断

进入「操作日志」页。

- 记录所有 Git 操作与账号验证、仓库扫描、API 错误等。
- **筛选**：按操作类型、状态、仓库、时间范围。
- **详情**：点行打开详情，显示完整命令、输出、错误(均已脱敏)，以及命中的**中文翻译提示**。
- **清理旧日志**：顶部按钮触发危险操作确认，按指定天数清理。

常见错误中文提示覆盖：认证失败、仓库不存在、SSH 公钥拒绝、DNS 解析失败、目标目录冲突、非快进推送、TLS 连接中断等。

## 8. 设置中心

进入「设置」页，分五个标签：

- **通用**：启动时打开上次仓库、启动时自动检查仓库状态、默认项目目录、默认 Clone 协议、默认并发数、默认目录组织方式、语言(中/英)、主题(浅色/深色/跟随系统)。标签底部另有「应用数据目录」与「日志与存储」两个维护分区，详见 8.1 / 8.2。
- **Git**：Git 可执行文件路径(含「检测 Git」按钮)、user.name、user.email、默认 pull/push 策略。以上四项保存后即在实际 Git 操作中生效——自定义路径作用于 fetch/pull/push/commit 等全部命令，pull/push 策略决定合并/推送方式，user.name/email 在仓库尚无身份时自动写入仓库级配置。
- **网络**：HTTP/HTTPS 代理、使用系统代理、API 超时、Clone 超时。
- **外部工具**：默认编辑器、终端、文件管理器，以及用 VS Code/Cursor/JetBrains 打开仓库的命令。
- **账号与安全**：逐个账号显示**凭据状态**(「已存储」绿标 / 「凭据缺失」红标)，但**绝不显示 Token 明文**。
  - **重新验证**：输入新 Token → 测试连接 → 成功后保存到密钥库(修复缺失或失效的凭据)。
  - **删除凭据**：危险操作确认后删除密钥库中的 Token，但保留账号记录，可日后重新验证补回。

### 8.1 应用数据目录

「通用」标签底部的「应用数据目录」分区管理 GitView 数据库与日志的存放位置：

- **当前数据目录**：只读展示当前实际使用的目录。
- **更改目录**：选择新目录并二次确认后，把数据库与日志**复制**到新目录；**需重启应用才生效**，且重启前发生的改动不会进入新目录。迁移后旧目录会保留。
- **删除旧目录**：迁移完成后出现，二次确认后**永久删除**旧数据目录(不可恢复)。
- **恢复默认目录**：把数据复制回应用默认位置(若默认位置已有旧数据会被覆盖)，同样需重启才生效。

### 8.2 日志与存储

「通用」标签底部的「日志与存储」分区展示并清理**磁盘上的日志文件**：

- 显示日志目录路径、占用大小、日志文件数，点「刷新」可重新统计。
- **清理历史日志**：二次确认后删除**当天之前**的所有日志文件、仅保留当天，并提示释放的空间。

> 注意：这里清理的是磁盘日志**文件**；与 §7「操作日志」页的「清理旧日志」(按天数清理数据库中的**操作记录**)是两个独立功能，互不影响。

所有设置重启后保持生效。主题切换即时预览。

## 9. 安全说明

GitView 在设计上把凭据安全作为硬性要求：

- **Token 存储**：只存入操作系统原生密钥库，**绝不**写入数据库明文字段、日志或界面。
- **日志脱敏**：所有命令、输出、错误在落库前统一脱敏 Token、密码、带 token 的 URL。
- **HTTPS 克隆**：Token 经临时凭据脚本注入，不进入 `remote.origin.url`；任务结束立即清理临时脚本，应用崩溃遗留的临时脚本会在下次启动时自动清理。
- **删除二次确认**：删除账号 / 凭据、丢弃变更、移除本地仓库、清空 Clone 任务、清理旧日志等破坏性操作均需经确认对话框(部分需输入关键词)。
- **自签名证书**：仅对你显式勾选的那个自建 GitLab 实例生效，不影响全局。

## 10. 常见问题(FAQ)

**Q：启动后一直停在「未检测到 Git」引导页？**
A：确认已安装 Git 且在 PATH 中；或点「去设置手动指定路径」填写 git 可执行文件的完整路径，再回引导页「重新检测」。

**Q：账号显示「凭据缺失」？**
A：密钥库中的 Token 丢失(如手动清理过 keyring)。到「设置 → 账号与安全」点「重新验证」重新输入 Token。

**Q：Push 报「remote has new commits / non-fast-forward」？**
A：远程有你本地没有的提交，先 Pull 合并后再 Push。

**Q：Pull 后提示有冲突？**
A：GitView V1 不内置冲突解决，请用外部工具(VS Code / `git mergetool`)解决后提交。

**Q：自建 GitLab 报证书错误？**
A：在该账号高级设置勾选「允许自签名证书」后重新测试连接(仅对该实例生效)。

**Q：内网 HTTP 地址保存被拦？**
A：HTTP 明文连接会弹安全提示，确认后才允许保存。

**Q：国内访问 GitHub 慢 / 失败？**
A：在「设置 → 网络」配置 HTTP/HTTPS 代理或启用系统代理；V1 不提供内置加速。

**Q：macOS/Windows 打不开下载的安装包？**
A：V1 未做公证/签名。macOS 右键应用 →「打开」；Windows SmartScreen →「更多信息」→「仍要运行」。

**Q：批量克隆里某个任务变成 skipped？**
A：目标目录已存在同名仓库目录，GitView 默认跳过以免覆盖。换目录或清理后重试。
