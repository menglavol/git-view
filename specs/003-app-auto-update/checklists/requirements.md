# 需求质量核对清单

**Feature**: 003-app-auto-update | 关联 [spec.md](../spec.md)

## 需求完整性

- [x] 更新来源明确（GitHub Releases `Menglavoll/git-view`）
- [x] 更新方式明确（应用内自动下载安装，用户已确认）
- [x] UI 落点明确（设置 → 通用 Tab 底部「关于与更新」分区，用户已确认）
- [x] 覆盖三平台（macOS / Windows / Linux）兼容格式差异
- [x] 失败路径有定义（网络失败 / 无 Release / 验签失败 / 限流）

## 边界与非功能

- [x] 明确非目标：不做 OS 代码签名/公证（仅 updater 包签名）

- [x] 明确安全约束：minisign 验签防篡改、私钥不入库
- [x] 明确失败不阻断：检查失败不影响设置页其余功能
