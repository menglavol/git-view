# Feature Specification: 应用内自动检查与更新

**Feature Branch**: `003-app-auto-update`
**Created**: 2026-07-13
**Status**: Draft（待用户批准 plan 后进入实施）
**Input**: 用户「在设置中添加检查软件更新功能，更新来源为本项目 GitHub」+ 已确认采用「应用内自动下载安装」路径、UI 落在通用 Tab 底部、检查仓库为 `Menglavoll/git-view`

## 概述

为 GitView 桌面客户端提供**应用内自动更新**能力：用户在设置中心手动触发检查，客户端从本项目 GitHub Releases 获取最新版本信息；若有新版，展示版本号与发布说明，用户确认后由应用自动下载更新包、校验签名、安装并重启到新版本，全程无需手动前往网页下载。

## 用户场景

### 主场景：检查并安装新版本

1. 用户打开「设置 → 通用」，滚动到底部「关于与更新」分区，看到**当前版本号**。
2. 点击「检查更新」。
3. 客户端请求 GitHub 更新清单，比对版本：
   - **有新版**：显示「发现新版本 vX.Y.Z」、发布说明（可折叠）、「下载并安装」与「稍后」按钮。
   - **已最新**：显示「已是最新版本」。
   - **失败**：显示中文友好错误提示，设置页其余功能不受影响。
4. 用户点「下载并安装」→ 显示实时下载进度条。
5. 下载完成并验签通过后，提示「更新已就绪，是否立即重启」。
6. 用户点「立即重启」→ 应用重启进入新版本；点「稍后」→ 下次启动应用时生效。

### 边界场景

- **无正式 Release / 仅草稿**：清单不可达，提示「暂无可用更新」而非报错。
- **网络失败 / API 限流**：捕获并给中文提示，不阻断。
- **验签失败**（更新包被篡改或签名不匹配）：拒绝安装，提示更新校验失败。
- **Linux `.deb` 用户**：`.deb` 不支持原地更新，提示前往 Release 手动下载（仅 AppImage 支持自动更新）。
- **未做 OS 代码签名**：安装/首次运行仍触发系统安全警告，属已知现状，非本特性缺陷。

## 功能需求

- **FR-001**：设置「通用」Tab 底部 MUST 提供「关于与更新」分区，展示应用当前版本号（取自应用元信息，不硬编码）。
- **FR-002**：MUST 提供「检查更新」操作，向本项目 GitHub Releases 更新清单发起请求。
- **FR-003**：检查结果 MUST 区分「有新版 / 已最新 / 失败」三态并给出对应中文反馈。
- **FR-004**：有新版时 MUST 展示目标版本号与发布说明（发布说明可折叠、过长可截断）。
- **FR-005**：用户确认后 MUST 支持应用内下载更新包并显示实时进度。
- **FR-006**：下载的更新包 MUST 经签名校验（minisign），校验失败 MUST 拒绝安装并提示。
- **FR-007**：安装完成后 MUST 引导用户重启以应用新版本，支持「立即重启 / 稍后」。
- **FR-008**：检查/下载失败 MUST 不阻断设置页其余功能（失败仅提示策略）。
- **FR-009**：所有面向用户的文案 MUST 走 i18n（中/英）。
- **FR-010**：更新来源 MUST 为 `Menglavoll/git-view` 的 GitHub Releases，客户端不得请求用户账号平台或携带用户 Token。
- **FR-011**：发布流程 MUST 在发版时为各平台安装包生成签名并产出更新清单（`latest.json`），使客户端可检测到新版本。

## 非功能需求

- **NFR-001（安全）**：签名私钥 MUST NOT 入库，仅存 GitHub Secrets 与安全备份；公钥可明文入配置。
- **NFR-002（安全）**：更新请求为匿名只读，MUST NOT 泄露用户任何凭据。
- **NFR-003（跨平台）**：macOS / Windows / Linux(AppImage) MUST 可自动更新；Linux deb 降级为手动提示。
- **NFR-004（质量）**：新增/修改源文件中文注释比例 ≥ 0.3；无调试输出；单文件 ≤500 行、函数 ≤50 行。

## 明确不做（本特性范围外）

- OS 代码签名与公证（Apple 证书 / Windows 代码签名）——独立议题，另行规划。
- 自动/后台静默更新与更新提醒推送——V1 仅手动触发。
- 增量/差分更新——采用整包替换（updater 默认）。
- 更新渠道（稳定/测试通道）切换——后续版本考虑。
- 回滚到旧版本——不提供。

## 假设与依赖

- 依赖 Tauri 官方 `tauri-plugin-updater` 与 `tauri-plugin-process`。
- 依赖 `tauri-action` 在 CI 发版时自动生成签名与 `latest.json`。
- 需先完成一次带签名的基线发版，之后老版本才能检测到更新。
- 私钥生成与 GitHub Secrets 配置由仓库维护者（用户）在 GitHub 后台执行。

## 验收标准

1. 设置通用 Tab 底部可见「关于与更新」，正确显示当前版本号。
2. 无新版时点检查，提示「已是最新」。
3. 存在更高版本 Release 时点检查，展示新版本号与发布说明。
4. 点「下载并安装」显示进度，完成后引导重启，重启后版本号更新为新版。
5. 篡改/错误签名的更新包被拒绝安装并提示。
6. 断网时检查给出中文提示且设置页其余功能正常。
7. 中英文切换下所有更新相关文案正确本地化。
8. `cargo clippy -D warnings`、`eslint --max-warnings 0`、注释比例门禁全绿。
